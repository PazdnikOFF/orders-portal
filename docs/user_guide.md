# Инструкция пользователя

## 1. Вход в систему

Откройте адрес портала и введите логин и пароль. Доступ возможен только после
авторизации. Через **45 минут** неактивности система автоматически завершит
сессию — потребуется войти снова.

## 2. Представления заказов

### Таблица (по умолчанию)

Список заказов с колонками: Менеджер, Дистрибьютор, Потенциальный пользователь,
Участники проекта, Комплект, Дата обращения, Прогнозируемая дата, Номер заказа,
Статус.

- **Поиск** — по номеру, менеджеру, организации, комплекту.
- **Фильтр по статусу** — выпадающий список.
- **Смена статуса прямо в таблице** — если ваша роль и стадия это позволяют.
- **Файл** — если к заказу прикреплён документ, в колонке «Номер заказа»
  отображается ссылка 📎 для просмотра.
- **Выбор строки** — открывает нижний блок с карточкой записи.

### Kanban

Вкладка **Kanban** — колонки «В плане», «В работе», «Произведён», «Отмена».
Карточка содержит номер, менеджера, дистрибьютора, потенциального пользователя и
прогнозируемую дату. Если у вас есть право смены статуса — карточки можно
**перетаскивать** между колонками. Иначе доска работает в режиме просмотра.

## 3. Нижний блок — карточка записи

По умолчанию свёрнут; кнопка **Свернуть/Развернуть** управляет видимостью. При
выборе строки в таблице карточка автоматически раскрывается и заполняется.

В карточке можно:

- просмотреть полные данные записи;
- отредактировать доступные поля (недоступные показаны заблокированными);
- загрузить документ заказа и открепить его;
- посмотреть историю изменений;
- сохранить изменения или отменить их.

## 4. Создание новой записи

Кнопка **+ Добавить заказ** (если у вас есть право создания). Обязательны все
поля, кроме файла:

| Поле | Как заполняется |
|---|---|
| Менеджер | выбор из справочника сотрудников (тип «Менеджер») |
| Дистрибьютор | ввод ИНН → «Проверить» → подгрузка наименования |
| Потенциальный пользователь | ввод ИНН → «Проверить» |
| Участники проекта (Комментарий) | список организаций по ИНН (+/✕) |
| Комплект | свободный текст |
| Прогнозируемая дата реализации | дата ДД.ММ.ГГГГ |
| Дата обращения | **заполняется системой** автоматически |
| Номер заказа | **присваивается системой** автоматически |
| Статус | по умолчанию «В плане» |

Организации подгружаются по ИНН: введите ИНН, нажмите «Проверить» — появится
наименование. Полные данные сохраняются в справочник при сохранении записи.

## 5. Работа с файлом заказа

- **Загрузить документ заказа** — файл сохраняется под номером заказа.
- **Просмотр** — ссылка 📎 в таблице или в карточке (только для авторизованных).
- **Удалить из карточки** — связь с записью снимается, но **файл физически не
  удаляется** (к имени добавляется «_», например `_ORD-000001.pdf`).

Допустимые типы и размер файла ограничены настройками системы.

## 6. Статусы

«В плане» → «В работе» → «Произведён», либо «Отмена». После перехода в
«Произведён» или «Отмена» редактирование записи запрещено (запись закрыта).

## 7. Роли и ограничения

- **Менеджер** видит только записи, где он указан как менеджер, и работает в
  режиме просмотра (включая файлы и историю).
- **Руководитель** видит все записи только для просмотра.
- **Оператор** просматривает все записи, создаёт и редактирует по правилам
  стадий.

Если поле заблокировано — его редактирование не разрешено вашей ролью или
текущей стадией заказа.
